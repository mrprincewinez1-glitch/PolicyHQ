# PolicyHQ Office Hours

Use this lightweight version when full GStack would be too expensive.

## Output Format

1. Problem
2. Target user
3. Smallest useful MVP
4. What not to build yet
5. Risks
6. Recommendation

## Decision Labels

- Build now
- Build later
- Research first
- Reject for MVP

Keep the answer concise. The goal is clarity before code, not a long essay.
