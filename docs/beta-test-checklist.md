# PolicyHQ Beta Test Checklist

Use this checklist when testing PolicyHQ with real agents. Do not explain too much while they use the app. Watch where they hesitate, click the wrong thing, or ask questions.

## Goal

Find out whether PolicyHQ helps agents manage clients, renewals, follow-ups, prospects, commissions, and relationships with less stress than their current offline process.

This is not a sales demo. It is a usability and demand test.

## Test Rules

- Use a laptop or phone similar to what the agent normally uses.
- Ask the agent to think out loud.
- Do not guide them unless they are completely stuck.
- Write down exact words they use.
- Note every confusing screen, button, label, or step.
- After the test, ask whether they would use it again next week.

## Agent Profile Notes

Record this before the test:

- Agent name:
- Agent type: Life / Non-life / Health / Mixed
- Years in insurance:
- Approximate number of clients:
- Current system: notebook / Excel / WhatsApp / insurer portal / memory / other
- Biggest pain today:
- Device used for test: phone / laptop / tablet

## Test 1: First Impression

Ask:

1. What do you think this app is for?
2. What would you click first?
3. Is anything confusing before you even start?

Observe:

- Do they understand the dashboard?
- Do they notice the important cards?
- Do they understand the sidebar?
- Do they understand the difference between Clients, Prospects, Policies, Commissions, and Renewal Alerts?

## Test 2: Client Book

Task:

1. Add one client manually.
2. Add phone number, date of birth if available, and basic contact details.
3. Find the client again from the Clients page.

Observe:

- Was adding a client easy?
- Did any field feel unnecessary?
- Did any important field feel missing?
- Could they find the client after saving?

Questions:

1. Would you enter your real clients here?
2. What would stop you from doing that?
3. Would import be more useful than manual entry?

## Test 3: Policy And Renewal Workflow

Best for non-life, health, or mixed agents.

Task:

1. Add or view a policy for a client.
2. Find policies expiring this week, next week, or this month.
3. Open a renewal-related card.
4. Use the WhatsApp or call action if available.

Observe:

- Can they find who needs attention?
- Do renewal labels make sense?
- Does the app reduce the need to remember dates manually?
- Does the WhatsApp action feel useful?

Questions:

1. How do you currently remember renewals?
2. What happens when you miss one?
3. Would this dashboard help you check renewals every Monday?

## Test 4: Prospects And Follow-Ups

Task:

1. Add one prospect.
2. Set a follow-up date.
3. Find follow-ups due today.
4. Use call or WhatsApp from the prospect card.

Observe:

- Does the prospect status make sense?
- Is follow-up date easy to set?
- Does the agent understand what to do next?

Questions:

1. How do you currently track prospects?
2. Do you lose prospects because you forget to follow up?
3. Would follow-up reminders make you use PolicyHQ more often?

## Test 5: Relationship Manager

Task:

1. Find birthdays or relationship check-ins.
2. Use a WhatsApp action for a client if available.

Observe:

- Does the agent care about birthday reminders?
- Do relationship check-ins feel useful or distracting?
- Does this make PolicyHQ feel more valuable than a spreadsheet?

Questions:

1. Do you regularly check in with existing clients?
2. Would birthday reminders help you maintain client relationships?
3. What other relationship moments should PolicyHQ track?

## Test 6: Life Agent Lapse Shield

Use this only with a life agent.

Task:

1. Explain the Lapse Shield idea in one sentence: "Upload your monthly commission statement and PolicyHQ helps show which clients may not have paid."
2. Ask what format their monthly statement comes in.
3. If they have a sample, inspect whether it includes policy numbers and client names.

Questions:

1. Do you receive your monthly commission statement as PDF, Excel, email, or portal view?
2. Can you download it?
3. Does it include policy numbers?
4. Does it include client names?
5. Do you currently check who is missing from the statement?
6. If PolicyHQ showed missing clients, would you contact them immediately?

Do not build more automation until this is confirmed with a real statement.

## Test 7: Import Flow

Use this if the agent already has a client list.

Task:

1. Ask them where their list currently lives.
2. Try importing a small sample if possible.
3. Ask them to map columns if needed.
4. Watch whether the import screen feels understandable.

Observe:

- Does the import accept the format they actually have?
- Are missing fields explained clearly?
- Does mapping columns feel too technical?
- Does the preview help them trust the import?

Questions:

1. Would import make you more likely to use PolicyHQ?
2. What file format do you usually have?
3. Would you need help preparing the file?

## Final Questions

Ask these at the end:

1. What was the most useful part?
2. What was confusing?
3. What felt unnecessary?
4. What is missing before you would use this with real clients?
5. Would you use this again next week?
6. Would you recommend it to another agent?
7. Would you pay for this? If yes, how much per month feels fair?

## Scoring

Use this simple score after the session:

- 5: Agent wants to use it immediately with real clients.
- 4: Agent sees clear value but needs small fixes.
- 3: Agent understands it but is not excited yet.
- 2: Agent is confused or prefers their current system.
- 1: Agent would not use it.

## Red Flags

Pay attention if:

- The agent cannot understand the dashboard without explanation.
- They do not trust entering client data.
- Import feels too hard.
- WhatsApp/call actions are not obvious.
- Renewal workload is not visible enough.
- Life agents do not have downloadable statements.
- The app feels like "more work" instead of "less remembering."

## Success Signal

The strongest positive signal is not praise. It is behavior.

Look for:

- "Can I add more clients?"
- "Send me the link."
- "Can this remind me every week?"
- "Can I use this for my real list?"
- "How much will this cost?"
